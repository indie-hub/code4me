# Code of Conduct

This project follows the [Contributor Covenant v2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) — the de-facto standard for open-source community guidelines. The full text lives at the link above; please read it before participating.

## In short

Everyone interacting with this project (contributors, reviewers, issue authors, discussion participants) is expected to engage constructively and respectfully. The Covenant's pledge is that participation is free from a long list of well-defined unacceptable behaviours; the link above enumerates them precisely.

## Reporting

If you observe a violation, open a [GitHub issue](../../issues/new/choose) using the **bug report** template, OR contact the maintainer privately via the repository's listed contact channel. The Covenant's "Enforcement Guidelines" describe the response process: maintainers will review, communicate consequences, and act within a reasonable timeframe.

When reporting, include:

- A brief description of what happened.
- Where it happened (issue link, discussion link, PR comment URL).
- Anyone you'd prefer not see the report (the maintainer keeps reports confidential to the extent reasonable for resolution).

## Scope

The Covenant applies to project spaces (issues, discussions, PRs, the project's documentation) AND public spaces when individuals are representing the project (e.g., posts under the project's name on social media).

## Maintainer responsibility

Maintainers may remove, edit, or reject contributions that violate the Covenant. Repeated or severe violations may result in a temporary or permanent ban from project participation. Decisions are at the maintainer's discretion and follow the Covenant's enforcement-guidelines tiers.
